# Future Direction — Context Governance

Este documento existe para preservar una decisión de diseño.
No implica trabajo inmediato ni compromisos de roadmap.

## Punto de partida validado
El MVP valida que:
- crear contexto explícito aporta valor
- el contexto versionado mejora la predictibilidad de la IA
- el usuario deja de reescribir prompts

## Evolución natural (post-MVP)
Una vez el contexto es confiable, el siguiente paso lógico es:

- aplicar reglas del sistema sobre ese contexto
- compilar contratos ejecutables para IA
- hacer el comportamiento de la IA predecible y auditable

## Principio clave
El usuario no define reglas.
El usuario elige comportamientos predefinidos por el sistema.
El sistema gobierna.
El agente obedece.

## Nota importante
Esta capa NO forma parte del MVP actual.
Solo se abordará si el Context Composer demuestra uso real y recurrente.
